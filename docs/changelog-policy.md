# Changelog Policy

These changelogs are the standard running record for work across threads:

- `docs/discovery-tech-changelog.md`
- `docs/ai-analysis-changelog.md`
- `docs/ui-ux-changelog.md`

Policy:
- Update the matching changelog before finishing meaningful work in that category.
- Keep entries brief, concrete, and dated.
- Include:
  - Summary
  - Why
  - Validation
  - Files
- Do not mix categories in the same changelog.
- Do not log unrelated repo housekeeping unless it directly affects that category.

Category guide:
- Discovery Tech: discovery pipeline, ATS extraction, seed selection, filtering, diagnostics, and discovery-related tests.
- AI Analysis: scoring, qualification, ranking, prompts, rationale generation, profile interpretation, and AI-analysis-related tests.
- UI/UX: interface behavior, layout, styling, interactions, usability, and UI-related tests.
